# 골동품 복원 공방 — Claude Code 인계 메모

## 이 프로젝트가 뭔가
파워워시 시뮬레이터 스타일의 **3D 골동품 세척·복원 게임**. 단일 HTML 파일(Three.js r128). 모바일 터치 + PC 마우스 모두 지원. 골동품 표면의 때를 도구로 문질러 닦고, 부품을 분해(나사 풀기 포함)하며 구석구석 청소하는 것이 핵심 재미.

## 넘기는 파일 2개
- **`antique-restoration.html`** (약 1730줄) — 본편. 4개 골동품(회중시계·청동향로·오르골·도자기주전자) 완성. 이걸 이어서 개선하면 됨.
- **`style-sample-censer.html`** (약 380줄) — 스타일 실험용 샘플. "밝고 둥근 무광 토이" 스타일로 향로 하나만 만든 것. 본편과 별개. 스타일 방향 참고용.

작업 시작할 때 브라우저로 두 파일을 각각 열어보면 현재 상태가 바로 보임.

## 본편 아키텍처 (antique-restoration.html)

전부 단일 `<script>` 안에 있음. 주요 시스템:

### 1. 오염/세척 시스템 (`class DirtSystem`)
- 파트별로 4겹 캔버스 텍스처 생성: albedo(색) + bump(요철) + roughness(광택) + metalness(금속성).
- 오염은 모래층(sand) + 먼지층(dust) 2겹. `drawSand`/`drawDust`는 **값 노이즈 기반 유기적 얼룩**(원형 폴카닷 아님).
- `brush(u,v,r,sandS,dustS)` — 레이캐스트로 얻은 UV 좌표에 지우개(destination-out)로 문질러 오염 제거. 도구별로 벗겨지는 층이 다름.
- 닦인 곳은 광택/금속성이 돌아와 경계가 또렷해짐.
- `wrap` 옵션('full'/'half'/'none')으로 원통 이음새 랩어라운드 처리.

### 2. UV 분리 (`packCylUVs`, `packBoxUVs`)
- 원통의 윗면/아랫면/옆면이 텍스처의 서로 다른 영역을 쓰도록 UV 재배치. **안 닦은 면이 같이 닦이는 버그의 해결책**이었음. 새 지오메트리 추가 시 반드시 적용.

### 3. 파트/분해 시스템
- `makePart(name, outPos, outRot)` → `addMesh(...)` 또는 `detail(...)`로 메시 부착 → `finishPart(ant, part, pos)`.
- `addMesh` = 오염 시스템 포함(닦임). `detail` = 순수 장식(안 닦임, 그림자만).
- 분해: 탭하면 `part.detached` 토글, `out` 위치로 애니메이션.
- **나사 잠금**: `addScrew(ant, host, pos, rot, outPos)` — host 파트를 잠금. 나사를 다 풀어야 host 분리 가능. `part.requires = otherPart`로 선행 분리 조건도 지정 가능(예: 무브먼트는 뒷뚜껑을 먼저 열어야 접근).

### 4. 도구 (파워워시식 3D 도구)
- 3종: blow(에어블로워)/brush(세척솔)/swab(면봉). 각각 **실제 3D 모델**(지오메트리 조합)이 청소 지점을 겨눔. 팁에서 분사 파티클(jetSpray).
- `TOOLS[tool].wr` = **월드 공간 반경**(픽셀 아님). `pxRadius()`가 삼각형의 월드:UV 비율로 픽셀 반경 환산 → 부품마다 체감 크기 일정.
- UV 점프(이음새/캡 경계) 시 이어그리기 차단 로직 있음.

### 5. 렌더링 (최근 업그레이드)
- 그림자(PCFSoftShadowMap), PMREM으로 구운 IBL 환경맵, UnrealBloomPass 후처리.
- 후처리 애드온은 jsdelivr CDN에서 로드(three@0.128.0 examples/js/). **로드 실패 시 기본 렌더로 폴백**하는 안전장치 있음(`if(composer)`).
- ACES 톤매핑. 유물은 허공에 뜬 느낌(받침대 없음, 아래 후광만).

### 6. UI
- 상단: 유물 선택 **드롭다운**(콤보박스). 완료 시 ✓ 체크.
- 하단: 회전/청소/분해 모드 + 현재도구 버튼(청소 모드만) + 진행률 명판.
- 도구는 **팝오버**로 선택 즉시 닫힘.
- 우측: 확대(＋)/축소(－)/오염표시 히트맵(🔍)/제자리(⟲).
- 중앙 힌트는 4.2초 후 자동 소멸, ? 버튼으로 재호출.
- 세척음/분해음/완료 팡파레는 WebAudio 실시간 합성(오디오 파일 없음).

### 7. 조작
- 회전 모드: 드래그 회전, 휠/핀치 줌. **상하 회전 제한 없음**(바닥까지 뱅그르르, 뒤집기 버튼은 제거됨).
- 청소 모드: 유물 위 문지르면 세척, 빈 곳/우클릭 드래그는 회전.
- 분해 모드: 부품 탭하면 분리.

## 각 골동품 구조

- **회중시계**: 케이스(열린 튜브)+다이얼+베젤 / 바늘(시·분·초, addMesh로 먼지 묻음, 개별 분리) / 뒷뚜껑(나사2) / 무브먼트 지판(나사3, requires=뒷뚜껑; 기어트레인 4단+태엽통+밸런스휠+헤어스프링+루비보석) / 용두. **주의: 좌표 배치가 까다로워 여러 번 수정함. 무브먼트가 케이스 안에 들어가고 뒷뚜껑 평면을 안 뚫도록 Y좌표 주의.**
- **청동 향로**: 몸통(내벽+바닥 있음)+뇌문 띠+고리손잡이 / 뚜껑(내부천장+연기구멍6) / 다리3.
- **오르골**: 본체(4벽+바닥 열린 상자) / 무브먼트(실린더드럼+핀+개별빗살20+태엽통+거버너+기어2, 나사2) / 뚜껑(경첩+거울판) / 서랍.
- **주전자**: 본체(내벽+바닥, 찻물얼룩)+주둥이+손잡이 / 뚜껑(내부걸림턱).

## 스타일 방향 (미해결 논의)
사용자는 참고 이미지(밝고 둥근 무광 "토이 수리 게임", 백제금동대향로)를 보여주며 **"오브젝트가 투박하다, 더 정교하고 디테일하게"**를 원함. 두 갈래로 논의됨:

1. **밝고 둥근 무광 스타일** → `style-sample-censer.html`이 그 실험. 세그먼트 높이고, 무광 파스텔, 밝은 조명, 알록달록 나사, 몰딩·리벳·굽 등 디테일 추가. (사용자 피드백: 배경 너무 밝음→낮춤, 오염 너무 옅음→진하게, 형태는 OK지만 디테일 부족→몰딩/리벳/굽 추가함.)
2. **실제 3D 모델(GLTF)** → 검토 결과: 이 웹 환경에선 bpy 불가, trimesh(+manifold3d)로 투각·subdivision·glb export는 가능함을 실증(구멍 뚫린 향로 몸통 .glb 생성 성공). 하지만 유기적 조각(용·인물)은 여전히 한계.
3. **Blender MCP** → 사용자가 이 방향에 관심. 공식 Blender 커넥터(Claude Desktop Customize→Connectors→Blender) 또는 오픈소스 ahujasid/blender-mcp. **이걸로 파트별(lid/body/legs) .glb를 만들어 GLTFLoader로 게임에 붙이는 게 최종 목표가 될 수 있음.**

## Claude Code에서 이어서 할 일 (우선순위)
1. 브라우저로 현재 게임 열어 상태 확인. (파일이 커서 통째 재작성 대신 부분 수정 권장.)
2. 스타일 방향 확정: 무광 토이 vs 사실적 골동품. 확정되면 4개 유물 전체에 일관 적용.
3. (선택) Blender 커넥터로 정교한 파트별 .glb 제작 → `GLTFLoader` 붙이기. 이때 오염 시스템은 glb의 UV에 맞춰 재연결 필요.
4. 회중시계 무브먼트 좌표 등 잔여 클리핑 최종 점검.

## 기술 메모 / 함정
- three.js **r128** 고정(CDN). `CapsuleGeometry`, `THREE.OrbitControls` 없음. `MathUtils`는 있음.
- 후처리 애드온은 `examples/js/`(UMD, 전역 THREE에 부착) 사용. 모듈 빌드 아님.
- 아주 복잡한 요청은 작은 단계로 쪼개서 진행(안정성).
- 파일이 단일 HTML이라 커짐. Code에서 모듈 분리를 고려해도 좋음(단, artifact 렌더 편의는 단일 파일이 나음).
- 브라우저 storage(localStorage 등) 사용 금지 이슈는 로컬 실행엔 무관하나, 다시 웹 아티팩트로 돌릴 거면 유의.
